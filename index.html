Visualization of Art Gallery Problem

We are developing a pedagogical application to visualize the Art gallery problem.
The Art gallery problem is the problem of determining the minimum number of guards required to guard an Art gallery room. The room is a simple polygon (polygon with no holes and no crossings), and the guards are a set of points inside, on the edges or on the vertices of the polygon. The guards inside the polygon placed at any point are called point guards, and the ones placed on the vertices are called vertex guards.

Finding the exact minimum number of point guards required to guard the entire room inside the room is a NP-hard problem. But, based on Steve Fisk's proof, we can find the sufficient number of vertex guards required to guard the polygonal room with n vertices in O(n log n) time. Fisk claims that to guard a simple polygonal room with n vertices, you never need more than floor(n/3) vertex guards.

We will be developing an application that visualizes this algorithm based on Fisk's proof.
The algorithm involves 2 major steps:

  1. Triangulation of the polygon
  2. 3-coloring of the triangulation of polygon

According to Fisk's proof, to guard the entire room with minimum vertex guards, the guards should be positioned on the vertices with the color that occurs the least number of times. The figures below show simple visualization of the Algorithm.

Features of the Tool:

  • Ability to draw a simple polygon with n vertices.
  • Ability to load a simple polygon from a text file (containing one point on each line, with x and y coordinates seperated by space)
  • Ability to detect incorrent input (polygon that is not simple)
  • Ability to get the result directly.
  • Ability to run the algorithm step-by-step automatically.
  • Ability to run the algorithm step-by-step manually with the option of going to previous step.
  • Ability to view different variations of the solution.
  • Ability to view the visibility of each guard in the solution.
  • Displays the sufficient number of vertex guards required to guard the polygonal room.

Challenges Faced:

  • Developing Vertex guard visibility algorithm that works for all possible test cases
  • Dealing with floating point precision
  • Running step by step visualization in PyGame

Issues:

  • Polygon Visibility is inaccurate in rare cases because of floating point comparisons (issues with precission).
  • Triangulation may fail sometimes if the points are placed too close to each other (bug in the ear clipping algorithm).
  • Does not always give minimum no. of guards (flaw in the approach, always finding the accurate answer is NP-hard problem).

Future Work:

  • Efficient and accurate Vertex guard visibility algorithm
  • Editing polygon during run time (by drag and drop of edges and vertices)
  • Ability to pause the visualization of the algorithm
  • More appealing visual effects
